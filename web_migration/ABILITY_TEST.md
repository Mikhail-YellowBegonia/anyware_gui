## 4. 动画状态机设计（60 FPS 严格执行）

动画分为 4 个连续的阶段，请利用 `requestAnimationFrame` 构建主循环，并维护清晰的状态机（Phase 1 ~ Phase 4）。

### 阶段 1：终端待机 (Terminal Idle)
- 画面中央显示提取自 SVG 的 `>` 和 `_` 图形。
- `>` 保持静止，常亮，纯白色。
- `_` 以 500ms 的固定频率闪烁（可见 -> 隐藏 -> 可见）。
- 持续时间：约 2 秒后进入下一阶段。

### 阶段 2：数据阵雨 (Data Rain & Shaping)
- `_` 停止闪烁，变为常亮。此时，它在视觉上将作为汉字“雨”字顶部的“横”笔画。
- 天空中开始生成随机的白色“雨滴”（短线段，粗细与 `>` 的笔画一致），以倾斜的角度（如左上到右下，夹角约 60 度）持续落下。
- **擦除逻辑（关键）**：如果雨滴的运动轨迹穿过了 `>` 的有效像素区域，请擦除 `>` 在该碰撞点上的像素（可利用 Canvas 的 `globalCompositeOperation = 'destination-out'` 结合线条绘制来实现）。随着雨滴不断冲刷，`>` 最终将被完全“洗去”。
- **塑形逻辑（关键）**：在 SVG 中，“雨”字的下方有 5 个垂直的笔画/点。请提取它们的坐标作为“靶点”。当倾斜的雨滴恰好穿过这些靶点坐标时：
   1. 该雨滴停止下落。
   2. 它的角度瞬间从“倾斜”矫正为“垂直”。
   3. 它永久停留在该处，成为“雨”字的一个笔画。
- 阶段结束条件：`>` 完全消失，且 5 个靶点全部被雨滴填满，视觉上与 `_` 组合成了一个完整的汉字“雨”。

### 阶段 3：雨生百谷 (Ripple Reveal)
- 倾斜下雨的背景特效迅速减弱并停止（雨滴不再生成，画面中的雨滴继续落出屏幕外）。
- 已形成的“雨”字开始向左平滑移动（Translate），为右侧腾出空间。
- 天空中落下最后 3 滴特殊的雨滴，它们精准地击中 SVG 中“生”、“百”、“谷”三个字的中心坐标。
- **命中特效**：雨滴命中坐标的瞬间，以该点为圆心，爆发一个极简的白色同心圆波纹扩散动画（Ripple）。
- 波纹扩散扫过的区域，对应的汉字（“生”、“百”、“谷”）逐渐显现（Opacity 0 -> 1）。
- 此时，四个字按照 SVG 中的原始排版完美居中展示。
- **色彩突变**：此时，“雨”字中的某两个特定笔画，颜色平滑过渡为品牌点缀色（苹果绿）。
- 持续时间：静止展示约 1.5 秒后进入下一阶段。

### 阶段 4：谷雨初绽 (Reorganization & Subtitle)
- 汉字组合开始重构：将短语“雨 生 百 谷”精简为“谷 雨”。
- **交叉换位**：最右侧的“谷”字向左侧平滑移动，最左侧的“雨”字向右侧平滑移动。
- 在“谷”和“雨”相向运动并越过对方的过程中，中间的“生”、“百”二字透明度逐渐降低（Fade Out），最终完全消失。
- “谷”和“雨”最终在画面中心汇合，形成排版紧凑的“谷 雨”二字。
- **副标题入场**：在“谷雨”下方，一段纯文本 `UI Framework Project Koyuu` 从下方滑入（Slide Up）并淡入。
- 文本样式设定为：`UI Framework Project` 为纯白色，字号较小；`Koyuu` 为品牌点缀色（苹果绿）。
- 动画至此彻底定格。
